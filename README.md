# LangStudy

LangStudy is a modern full-stack foreign language learning application designed for interactive language acquisition. It features customizable difficulty control through fill-in-the-blank dialogue exercises and automated Ebbinghaus retention reviews.

---

## 🛠 Common Commands

The `Makefile` in the root directory wraps all development, setup, and deployment actions:

| Command | Description |
| :--- | :--- |
| `make prepare` | **First-Time Setup**: Copies the default config template to `.env`, installs frontend npm packages, downloads Go dependencies, and sets up a local Python virtual environment (`.venv`) for `edge-tts` synthesis. |
| `make dev` | **Local Development**: Runs both frontend and backend dev servers with hot reloading. Automatically orchestrates local Postgres & Redis Docker containers via testcontainers and executes database migrations. |
| `make build` | **Production Build**: Compiles frontend assets into static files under `dist`, and builds a size-compressed production Go binary under `backend/bin/server`. |
| `make clean` | **Clean Build Files**: Deletes compiled server binaries in `backend/bin/` and clears the frontend `dist` directory. |
| `make dev-clean` | **Reset Dev Environment**: Forcefully stops and deletes the Postgres & Redis dev containers, and cleans up all temporary audio files generated by `edge-tts`. |
| `make tidy` | Run Go module clean-up (`go mod tidy`). |
| `make lint` | Run static analysis check on Go packages (`golangci-lint run ./...`). |

## 🧑‍💻 Developer Onboarding Guide

Follow these steps to initialize your local environment and begin adding features:

### Step 1: Environment & Config Initialization

1. Make sure your Docker daemon (Docker Desktop or Colima) is active and running.
2. In the root directory, run the onboarding setup:
   ```bash
   make prepare
   ```
3. **Configure API Secrets**:
   - The setup command copies `backend/.env.example` into a new [backend/.env](file:///Users/peigen/Documents/dev/0xHardfork/LangStudy/backend/.env) file. This file is git-ignored and contains private runtime variables.
   - Edit [backend/.env](file:///Users/peigen/Documents/dev/0xHardfork/LangStudy/backend/.env) to specify your model API keys:
     ```env
     LLM_API_URL=https://generativelanguage.googleapis.com/v1beta/openai/chat/completions
     LLM_API_KEY=your-gemini-or-openai-api-key
     LLM_MODEL_NAME=gemini-2.5-flash
     ```
   - *Note: On startup, the Go backend will automatically detect these environment values and sync them into the `llm_configs` table (ID=1) in your database. No manual SQL changes are required.*

---

### Step 2: Implement a New Feature (Code Conventions)

The backend follows a strict **Feature-First Architecture**, and the frontend relies on modular, styled components. When creating a new feature, follow this checklist of file locations:

#### 1. Database Schema Migration
Create sequential migration scripts in the [backend/migrations/](file:///Users/peigen/Documents/dev/0xHardfork/LangStudy/backend/migrations) folder:
- **Create Files**: `00000x_create_<feature>.up.sql` and `00000x_create_<feature>.down.sql`
- **Example**:
  ```sql
  CREATE TABLE IF NOT EXISTS vocabularies (
      id         BIGSERIAL PRIMARY KEY,
      user_id    BIGINT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
      word       VARCHAR(128) NOT NULL,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
  ```

#### 2. Backend Domain Layers
Create a dedicated package under `backend/internal/` (e.g., `backend/internal/vocabulary/`). You **must** create these 4 standard files to isolate concerns:
- [NEW] `model.go`: Define GORM schemas and JSON payload Request/Response DTO contracts.
- [NEW] `store.go`: Declare and implement GORM database operations.
- [NEW] `service.go`: Declare and implement business logic rules.
- [NEW] `handler.go`: Define Gin endpoint controllers, decode parameters, and return uniform JSON wrappers using the `platform/response` utility.

#### 3. Register Routing & Dependencies
Open [backend/cmd/server/main.go](file:///Users/peigen/Documents/dev/0xHardfork/LangStudy/backend/cmd/server/main.go) to wire up the new package layers:
1. Initialize the Store: `vocabStore := vocabulary.NewStore(db)`
2. Initialize the Service: `vocabService := vocabulary.NewService(vocabStore)`
3. Initialize the Handler: `vocabHandler := vocabulary.NewHandler(vocabService)`
4. Register the public or authenticated endpoints:
   ```go
   authed.GET("/vocabularies", vocabHandler.List)
   authed.POST("/vocabularies", vocabHandler.Create)
   ```

#### 4. Frontend API Mapping & Components
1. **Network Mapping**: Create an API request mapper (e.g. `vocabulary.ts`) in the `frontend/src/services/` directory to fetch the Go backend routes.
2. **UI Component Styling**: Develop React widgets under the `frontend/src/components/` folder.
   - *Theme Guard: Make sure admin components are wrapped in Antd's `<ConfigProvider>` using `theme.darkAlgorithm` to ensure high contrast, keeping text labels completely readable against the dark slate-950 page background.*
