一、 后端：

必须遵守标准 Go 工程化规范，强制处理所有 error，绝对不允许使用 _ 忽略错误。




二、 前端：

TS 严格模式：禁止在代码中出现任何 any 类型。所有 Props、State 和接口返回数据都必须定义清晰的 interface 或 type。

函数式与 Hooks：前端必须纯用 Functional Components 和 React Hooks，禁止出现 Class Components。

样式纪律：纯靠 Tailwind CSS 实现样式，绝对禁止写内联样式（style={{...}}），禁止在组件里自己手写普通 CSS 类。

三、 输出格式铁律：

拒绝半成品：无论代码多长，每次输出的单个文件必须是完整的、可直接复制粘贴运行的。

禁止省略：绝对、严禁使用 // ...、// 此处省略、// 保持原有逻辑 这种愚蠢的占位符。只要写了，就必须写全。

不要添加任何注释，代码要可以自解释

不要在代码中包含中文
